Introduction

// qu'est-ce que MLX Python ?

La MinilibX est une petite bibliothèque graphique en C conçue pour les étudiants de l'école 42. Le wrapper Python expose toutes ses fonctionnalités via ctypes, en adaptant l'API C pour qu'elle soit plus naturelle en Python.

ℹ️ Différences avec l'API C Les fonctions qui utilisent des pointeurs de sortie (&width, &height…) retournent un tuple en Python au lieu de modifier des arguments.
Fenêtres
Créer, afficher et détruire des fenêtres graphiques avec titre personnalisé.
Images & pixels
Manipuler des images en mémoire pixel par pixel. Charger PNG et XPM.
Événements
Réagir aux clics souris, touches clavier, fermeture de fenêtre via des callbacks Python.
Boucle principale
Lancer et contrôler la boucle d'événements MLX de manière non-bloquante.

Placez les fichiers dans un dossier mlx/ à côté de votre script :

from mlx import Mlx

class App:
    def __init__(self):
        self.mlx = Mlx()
        self.mlx_ptr = self.mlx.mlx_init()
        self.win = self.mlx.mlx_new_window(
            self.mlx_ptr, 400, 300, "Ma fenêtre")

    def on_mouse(self, button, x, y, data):
        print(f"Clic bouton {button} en {x},{y}")

    def on_key(self, keycode, data):
        print(f"Touche pressée : {keycode}")

    def on_close(self, data):
        self.mlx.mlx_loop_exit(self.mlx_ptr)

    def run(self):
        self.mlx.mlx_string_put(
            self.mlx_ptr, self.win, 20, 20, 0xFFFFFFFF, "Hello MLX!")
        self.mlx.mlx_mouse_hook(self.win, self.on_mouse, None)
        self.mlx.mlx_key_hook(self.win, self.on_key, None)
        self.mlx.mlx_hook(self.win, 33, 0, self.on_close, None)
        self.mlx.mlx_loop(self.mlx_ptr)

App().run()
⚠️ Ctrl+C ne fonctionne pas pendant mlx_loop ! Pour tuer le programme depuis le terminal, utilisez Ctrl+\ (SIGQUIT). Ou branchez mlx_loop_exit sur le bouton de fermeture.

Initialisation

// démarrer et libérer MLX

mlx_init()
Initialise la connexion avec le serveur graphique. Doit être appelé en premier.
→ mlx_ptr (void*) ou None si échec
mlx_release(mlx_ptr)
Libère toutes les ressources MLX. À appeler en toute fin de programme, après avoir détruit fenêtres et images.
→ 0 si succès
Référence des paramètres
mlx_init() → void* | None
Ouvre la connexion au serveur graphique (XCB/Vulkan). Aucun paramètre requis. Retourne None en cas d'échec (serveur absent, bibliothèques manquantes).
ParamètreTypeRôle
*(aucun)*—Cette fonction ne prend aucun argument.
→ retourvoid* | NonePointeur de connexion MLX à passer à toutes les autres fonctions. None si initialisation échouée.
mlx_release(mlx_ptr) → int
Ferme la connexion et libère toutes les ressources internes MLX. Appelez cette fonction après avoir libéré toutes les images et fenêtres.
ParamètreTypeRôle
mlx_ptrvoid*Pointeur de connexion retourné par mlx_init(). Identifie la session graphique à fermer.
→ retourint0 si succès.
python

Fenêtres

// créer et manipuler des fenêtres

mlx_new_window(mlx_ptr, width, height, title)
Crée une nouvelle fenêtre graphique à l'écran.
→ win_ptr (void*) ou None si échec
mlx_clear_window(mlx_ptr, win_ptr)
Efface tout le contenu de la fenêtre (fond noir).
→ 0 si succès
mlx_pixel_put(mlx_ptr, win_ptr, x, y, color)
Dessine un pixel directement dans la fenêtre. Lent pour beaucoup de pixels — préférer les images.
→ 0 si succès
mlx_destroy_window(mlx_ptr, win_ptr)
Détruit une fenêtre et libère ses ressources associées.
→ 0 si succès
Référence des paramètres
mlx_new_window(mlx_ptr, width, height, title) → void* | None
ParamètreTypeRôle
mlx_ptrvoid*Pointeur de connexion retourné par mlx_init().
widthuintLargeur de la fenêtre en pixels. Les valeurs hors écran peuvent causer des comportements indéfinis.
heightuintHauteur de la fenêtre en pixels.
titlestrTitre affiché dans la barre de titre de la fenêtre. Encodé automatiquement en UTF-8.
→ retourvoid* | NoneHandle de fenêtre à passer aux autres fonctions. None si la création échoue.
mlx_pixel_put(mlx_ptr, win_ptr, x, y, color) → int
⚠️ Lent — chaque appel envoie une requête au serveur graphique. Pour de nombreux pixels, utilisez mlx_new_image + mlx_put_image_to_window.
ParamètreTypeRôle
mlx_ptrvoid*Connexion MLX.
win_ptrvoid*Fenêtre destination où le pixel sera dessiné.
xuintColonne du pixel. Origine (0,0) en haut à gauche. MLX ne vérifie pas les bornes.
yuintLigne du pixel. L'axe Y est positif vers le bas.
coloruintCouleur 32 bits encodée selon le format du système (voir section Format des couleurs). Typiquement 0xAARRGGBB.
→ retourint0 si succès.
mlx_clear_window / mlx_destroy_window(mlx_ptr, win_ptr) → int
ParamètreTypeRôle
mlx_ptrvoid*Connexion MLX.
win_ptrvoid*Fenêtre à effacer (clear) ou à détruire (destroy). Après destroy, ce pointeur est invalide.
→ retourint0 si succès.

avec classe 
class App:
    def __init__(self):
        self.mlx = Mlx()
        self.mlx_ptr = self.mlx.mlx_init()
        self.win = self.mlx.mlx_new_window(
            self.mlx_ptr, 800, 600, "Mon app")

    def draw_pixel(self, x, y, color):
        self.mlx.mlx_pixel_put(self.mlx_ptr, self.win, x, y, color)

    def clear(self):
        self.mlx.mlx_clear_window(self.mlx_ptr, self.win)

    def cleanup(self):
        self.mlx.mlx_destroy_window(self.mlx_ptr, self.win)
        self.mlx.mlx_release(self.mlx_ptr)
ℹ️ Origine des coordonnées Le point (0, 0) est en haut à gauche de la fenêtre. L'axe Y descend vers le bas. MLX ne vérifie pas si x/y sont dans les limites — un pixel hors-fenêtre peut provoquer un crash.

Images

// pixels en mémoire + chargement fichiers

Les images permettent de modifier des pixels en mémoire puis d'afficher le tout en une seule fois — bien plus rapide que mlx_pixel_put appel par appel.

mlx_new_image(mlx_ptr, width, height)
Crée une image vide en mémoire (non initialisée).
→ img_ptr ou None
mlx_get_data_addr(img_ptr)
⚠️ API modifiée : retourne un tuple (data, bpp, size_line, format) au lieu d'utiliser des pointeurs.
→ (memoryview, int, int, int)
mlx_put_image_to_window(mlx_ptr, win_ptr, img_ptr, x, y)
Affiche l'image dans la fenêtre à la position (x, y).
→ 0 si succès
mlx_png_file_to_image(mlx_ptr, filename)
⚠️ API modifiée : retourne (img_ptr, width, height).
→ (img_ptr, int, int)
mlx_xpm_file_to_image(mlx_ptr, filename)
⚠️ API modifiée : retourne (img_ptr, width, height).
→ (img_ptr, int, int)
mlx_destroy_image(mlx_ptr, img_ptr)
Libère la mémoire de l'image. À appeler avant mlx_release.
→ 0 si succès
Référence des paramètres
mlx_new_image(mlx_ptr, width, height) → void* | None
ParamètreTypeRôle
mlx_ptrvoid*Connexion MLX.
widthuintLargeur de l'image en pixels. Doit correspondre aux accès au buffer.
heightuintHauteur de l'image en pixels. Stocké en interne pour calculer la taille du buffer.
→ retourvoid* | NoneHandle d'image. None si la création échoue.
mlx_get_data_addr(img_ptr) → (memoryview, int, int, int)
Accède au buffer brut de pixels de l'image. Le tuple retourné contient tout ce qu'il faut pour écrire des pixels manuellement.
ParamètreTypeRôle
img_ptrvoid*Image créée via mlx_new_image, mlx_png_file_to_image ou mlx_xpm_file_to_image.
→ datamemoryviewBuffer de pixels modifiable. Écrivez data[offset:offset+4] pour changer un pixel.
→ bppintBits par pixel (généralement 32). Divisez par 8 pour obtenir les octets par pixel.
→ size_lineintTaille d'une ligne en octets. Peut être supérieur à width×4 (padding). Utilisez offset = y * sl + x * (bpp // 8).
→ formatint0 = B8G8R8A8 (typique little-endian) ; 1 = A8R8G8B8. Détermine l'ordre des octets de couleur.
mlx_put_image_to_window(mlx_ptr, win_ptr, img_ptr, x, y) → int
ParamètreTypeRôle
mlx_ptrvoid*Connexion MLX.
win_ptrvoid*Fenêtre destination.
img_ptrvoid*Image à afficher. Peut être une image vide, PNG ou XPM.
xintPosition horizontale du coin supérieur gauche de l'image dans la fenêtre.
yintPosition verticale du coin supérieur gauche de l'image dans la fenêtre.
→ retourint0 si succès.
mlx_png_file_to_image / mlx_xpm_file_to_image(mlx_ptr, filename) → (img_ptr, width, height)
ParamètreTypeRôle
mlx_ptrvoid*Connexion MLX.
filenamestrChemin vers le fichier image (relatif ou absolu). Encodé automatiquement en UTF-8.

Créer une image et modifier ses pixels

python
sans classe 
# 1. Créer une image 200×200
img = m.mlx_new_image(mlx_ptr, 200, 200)

# 2. Obtenir l'accès au buffer de pixels
data, bpp, sl, fmt = m.mlx_get_data_addr(img)
#   data = buffer modifiable
#   bpp  = bits par pixel (32)
#   sl   = octets par ligne (peut être > width*4)
#   fmt  = 0: B8G8R8A8 | 1: A8R8G8B8

# 3. Écrire un pixel rouge en (10, 20)
x, y = 10, 20
offset = y * sl + x * (bpp // 8)
data[offset:offset+4] = (0xFFFF0000).to_bytes(4, 'little')

# 4. Afficher l'image dans la fenêtre
m.mlx_put_image_to_window(mlx_ptr, win, img, 0, 0)

# 5. Libérer quand terminé
m.mlx_destroy_image(mlx_ptr, img)
python
avec classe 
class App:
    def __init__(self, w, h):
        self.mlx = Mlx()
        self.mlx_ptr = self.mlx.mlx_init()
        self.win = self.mlx.mlx_new_window(self.mlx_ptr, w, h, "App")
        self.w, self.h = w, h

        # Créer et stocker l'image dans l'instance
        self.img = self.mlx.mlx_new_image(self.mlx_ptr, w, h)
        self.data, self.bpp, self.sl, self.fmt = \
            self.mlx.mlx_get_data_addr(self.img)

    def put_pixel(self, x, y, color):
        # Méthode de dessin sécurisée avec vérification des bornes
        if 0 <= x < self.w and 0 <= y < self.h:
            offset = y * self.sl + x * (self.bpp // 8)
            self.data[offset:offset+4] = color.to_bytes(4, 'little')

    def flush(self):
        # Envoyer le buffer vers la fenêtre
        self.mlx.mlx_put_image_to_window(
            self.mlx_ptr, self.win, self.img, 0, 0)

    def cleanup(self):
        self.mlx.mlx_destroy_image(self.mlx_ptr, self.img)
        self.mlx.mlx_destroy_window(self.mlx_ptr, self.win)
        self.mlx.mlx_release(self.mlx_ptr)

app = App(800, 600)
app.put_pixel(10, 20, 0xFFFF0000)  # rouge en (10,20)
app.flush()

Événements

// clavier, souris, boucle

mlx_loop(mlx_ptr)
Lance la boucle d'événements. Bloquant — retourne uniquement après mlx_loop_exit().
→ int (0 après exit)
mlx_loop_exit(mlx_ptr)
Demande l'arrêt propre de la boucle.
→ int
mlx_mouse_hook(win_ptr, callback, param)
Enregistre un callback pour les clics souris.
callback(button, x, y, param)
→ int
mlx_key_hook(win_ptr, callback, param)
Enregistre un callback pour les touches clavier (déclenché au relâchement).
callback(keycode, param)
→ int
mlx_expose_hook(win_ptr, callback, param)
Callback appelé quand la fenêtre doit être redessinée (Expose).
callback(param)
→ int
mlx_loop_hook(mlx_ptr, callback, param)
Callback appelé en continu à chaque itération de la boucle (game loop).
callback(param)
→ int
mlx_hook(win_ptr, x_event, x_mask, callback, param)
Hook générique pour n'importe quel événement X11. Signature du callback variable selon x_event.
→ int
Référence des paramètres
mlx_loop(mlx_ptr)  /  mlx_loop_exit(mlx_ptr)
mlx_loop démarre la boucle bloquante. mlx_loop_exit demande son arrêt depuis n'importe quel callback.
ParamètreTypeRôle
mlx_ptrvoid*Connexion MLX initialisée par mlx_init().
mlx_mouse_hook(win_ptr, callback, param) → int
ParamètreTypeRôle
win_ptrvoid*Fenêtre sur laquelle surveiller les clics.
callbackcallableFonction appelée à chaque clic : callback(button: int, x: int, y: int, param). Passez None pour désactiver.
paramanyDonnée utilisateur transmise telle quelle au callback. Utile pour passer un objet d'état.
mlx_key_hook(win_ptr, callback, param) → int
Déclenché au KeyRelease, pas au KeyPress. Pour le KeyPress, utilisez mlx_hook(win, 2, 1, ...).
ParamètreTypeRôle
win_ptrvoid*Fenêtre à surveiller.
callbackcallableFonction appelée : callback(keycode: int, param). Le keycode est le code X11 de la touche relâchée.
paramanyDonnée utilisateur transmise au callback.
mlx_loop_hook(mlx_ptr, callback, param) → int
Le callback est appelé à chaque itération de la boucle quand il n'y a pas d'événement en attente. Idéal pour la game loop (mise à jour + rendu).
ParamètreTypeRôle
mlx_ptrvoid*Connexion MLX.
callbackcallableFonction appelée continuellement : callback(param).
paramanyDonnée utilisateur (typiquement votre objet d'état global).
mlx_hook(win_ptr, x_event, x_mask, callback, param) → int
Hook générique. La signature du callback dépend de x_event.
ParamètreTypeRôle
win_ptrvoid*Fenêtre sur laquelle écouter l'événement.
x_eventuintNuméro d'événement X11 : 2=KeyPress, 3=KeyRelease, 4=ButtonPress, 5=ButtonRelease, 6=MotionNotify, 33=WM_DELETE_WINDOW…
x_maskuintMasque d'événement X11. Utilisez 0 pour WM_DELETE_WINDOW et les événements sans masque dédié.
callbackcallableSignature variable : event 2/3 → (keycode, param) ; event 4/5 → (button, x, y, param) ; event 6 → (x, y, param) ; autres → (param).
paramanyDonnée utilisateur.

Animation

// boucle de rendu, temps, mouvement

Une animation MLX repose sur une idée simple : à chaque frame, on efface le buffer, on dessine la nouvelle scène, puis on l'affiche. La boucle est orchestrée par mlx_loop_hook.

⏱️ Delta time

Temps écoulé depuis la dernière frame. Multipliez les vitesses par le delta pour que le mouvement soit indépendant du FPS de la machine.

🔁 Double buffer

On dessine dans un buffer data en mémoire, puis on l'envoie d'un coup avec mlx_put_image_to_window. Évite le scintillement (tearing).

🧹 Clear chaque frame

Avant de dessiner, remplissez le buffer en noir avec un memset-like. Sans ça, les frames précédentes restent visibles.

🔒 mlx_sync

Appelez mlx_sync(mlx_ptr, SYNC_IMAGE_WRITABLE, img) avant de modifier le buffer si l'image est en cours de rendu.

Fonctions clés pour l'animation
mlx_loop_hook(mlx_ptr, callback, param) → int
C'est la fonction centrale de l'animation. Le callback est appelé à chaque itération de la boucle d'événements quand aucun événement n'est en attente. C'est ici que se placent la mise à jour de l'état et le rendu.
ParamètreTypeRôle dans l'animation
mlx_ptrvoid*Connexion MLX. Identifie la session à laquelle rattacher la boucle.
callbackcallableVotre fonction de rendu : callback(param). Appelée en continu — c'est votre game loop. Doit être rapide (éviter les boucles Python longues).
paramanyObjet d'état passé à chaque appel. Typiquement un dict ou un objet qui contient position, vitesse, temps, etc.
→ retourint0 si succès. Un seul mlx_loop_hook actif à la fois par connexion MLX.
time.time() → float
Retourne le timestamp courant en secondes (float). La soustraction de deux appels donne le delta time entre deux frames.
PatternValeur typiqueUsage
delta = now - last0.016 sDurée d'une frame à 60 FPS. Multipliez vitesse × delta pour un mouvement fluide.
elapsed = now - start0…∞ sTemps total depuis le lancement. Utile pour les oscillations : sin(elapsed * vitesse).
fps = 1.0 / delta60, 120…FPS courant. Peut servir à l'afficher ou à limiter le rendu.

1 — Structure de base d'une animation

Le minimum pour avoir un rendu qui tourne en boucle avec gestion du temps.

python — boucle de base
sans classe 
import time
from mlx import Mlx

W, H = 800, 600

m = Mlx()
mlx_ptr = m.mlx_init()
win = m.mlx_new_window(mlx_ptr, W, H, "Animation")
img = m.mlx_new_image(mlx_ptr, W, H)
data, bpp, sl, _ = m.mlx_get_data_addr(img)

# État global de l'animation
state = {
    't_last': time.time(),   # timestamp de la frame précédente
    'elapsed': 0.0,          # temps total écoulé
    'x': W // 2,              # position X courante
    'vx': 200,                # vitesse en px/seconde
}

def put_pixel(x, y, color):
    if 0 <= x < W and 0 <= y < H:
        offset = y * sl + x * (bpp // 8)
        data[offset:offset+4] = color.to_bytes(4, 'little')

def clear_buffer():
    # Remplir tout le buffer en noir
    for i in range(0, sl * H, 4):
        data[i:i+4] = (0xFF000000).to_bytes(4, 'little')

def render(s):
    now = time.time()
    dt = now - s['t_last']      # delta time en secondes
    s['t_last'] = now
    s['elapsed'] += dt

    # ── Mettre à jour l'état ──────────────────
    s['x'] += s['vx'] * dt       # mouvement indépendant du FPS
    if s['x'] >= W or s['x'] <= 0:  # rebond sur les bords
        s['vx'] *= -1

    # ── Effacer le buffer ─────────────────────
    clear_buffer()

    # ── Dessiner un carré 10×10 ───────────────
    cx = int(s['x'])
    for dy in range(-5, 5):
        for dx in range(-5, 5):
            put_pixel(cx + dx, H // 2 + dy, 0xFF00E5A0)

    # ── Envoyer le buffer vers la fenêtre ─────
    m.mlx_put_image_to_window(mlx_ptr, win, img, 0, 0)

m.mlx_loop_hook(mlx_ptr, render, state)
m.mlx_hook(win, 33, 0,
    lambda d: m.mlx_loop_exit(mlx_ptr), None)
m.mlx_loop(mlx_ptr)

import time
from mlx import Mlx

class AnimApp:
    W, H = 800, 600

    def __init__(self):
        self.mlx = Mlx()
        self.mlx_ptr = self.mlx.mlx_init()
        self.win = self.mlx.mlx_new_window(
            self.mlx_ptr, self.W, self.H, "Animation")
        self.img = self.mlx.mlx_new_image(
            self.mlx_ptr, self.W, self.H)
        self.data, self.bpp, self.sl, _ = \
            self.mlx.mlx_get_data_addr(self.img)

        # État de l'animation
        self.t_last  = time.time()
        self.elapsed = 0.0
        self.x  = self.W / 2
        self.vx = 200         # px/seconde

    def put_pixel(self, x, y, color):
        if 0 <= x < self.W and 0 <= y < self.H:
            off = y * self.sl + x * (self.bpp // 8)
            self.data[off:off+4] = color.to_bytes(4, 'little')

    def clear(self):
        for i in range(0, self.sl * self.H, 4):
            self.data[i:i+4] = (0xFF000000).to_bytes(4, 'little')

    def tick(self):
        # Calcul du delta time
        now = time.time()
        dt = now - self.t_last
        self.t_last = now
        self.elapsed += dt
        return dt

    def update(self, dt):
        # Mettre à jour la physique
        self.x += self.vx * dt
        if self.x >= self.W or self.x <= 0:
            self.vx *= -1

    def draw(self):
        # Dessiner un carré vert
        cx = int(self.x)
        for dy in range(-5, 5):
            for dx in range(-5, 5):
                self.put_pixel(cx + dx, self.H // 2 + dy, 0xFF00E5A0)

    def render(self, _):
        dt = self.tick()
        self.update(dt)
        self.clear()
        self.draw()
        self.mlx.mlx_put_image_to_window(
            self.mlx_ptr, self.win, self.img, 0, 0)

    def run(self):
        self.mlx.mlx_loop_hook(self.mlx_ptr, self.render, None)
        self.mlx.mlx_hook(self.win, 33, 0,
            lambda d: self.mlx.mlx_loop_exit(self.mlx_ptr), None)
        self.mlx.mlx_loop(self.mlx_ptr)

2 — Balle rebondissante avec gravité

Simulation physique simple : gravité, rebond sur le sol, perte d'énergie.

python — balle rebondissante
sans classe 
import time, math
from mlx import Mlx

W, H = 800, 600
GRAVITY = 900    # px/s²
RADIUS  = 20     # rayon de la balle
BOUNCE  = 0.75   # coefficient de restitution (0=mou, 1=parfait)

m = Mlx()
mlx_ptr = m.mlx_init()
win = m.mlx_new_window(mlx_ptr, W, H, "Balle")
img = m.mlx_new_image(mlx_ptr, W, H)
data, bpp, sl, _ = m.mlx_get_data_addr(img)

state = {
    't': time.time(),
    'x': W / 2,  'y': H / 4,
    'vx': 150,   'vy': 0,
}

def put_pixel(x, y, c):
    if 0 <= x < W and 0 <= y < H:
        o = y * sl + x * 4
        data[o:o+4] = c.to_bytes(4, 'little')

def draw_circle(cx, cy, r, color):
    # Algorithme du cercle rempli
    for dy in range(-r, r + 1):
        dx = int(math.sqrt(r*r - dy*dy))
        for i in range(-dx, dx + 1):
            put_pixel(cx + i, cy + dy, color)

def render(s):
    now = time.time()
    dt  = min(now - s['t'], 0.05)  # clamper le dt max à 50ms
    s['t'] = now

    # Physique : gravité
    s['vy'] += GRAVITY * dt
    s['x']  += s['vx'] * dt
    s['y']  += s['vy'] * dt

    # Rebonds sur les murs
    if s['x'] + RADIUS >= W: s['x'] = W - RADIUS; s['vx'] *= -BOUNCE
    if s['x'] - RADIUS <= 0: s['x'] = RADIUS;     s['vx'] *= -BOUNCE
    if s['y'] + RADIUS >= H: s['y'] = H - RADIUS; s['vy'] *= -BOUNCE
    if s['y'] - RADIUS <= 0: s['y'] = RADIUS;     s['vy'] *= -BOUNCE

    # Clear + dessin
    for i in range(0, sl * H, 4):
        data[i:i+4] = (0xFF0A0C10).to_bytes(4, 'little')
    draw_circle(int(s['x']), int(s['y']), RADIUS, 0xFF7C6AF7)

    m.mlx_put_image_to_window(mlx_ptr, win, img, 0, 0)

m.mlx_loop_hook(mlx_ptr, render, state)
m.mlx_hook(win, 33, 0, lambda d: m.mlx_loop_exit(mlx_ptr), None)
m.mlx_loop(mlx_ptr)

import time, math
from mlx import Mlx

class Ball:
    GRAVITY = 900
    RADIUS  = 20
    BOUNCE  = 0.75

    def __init__(self, w=800, h=600):
        self.W, self.H = w, h
        self.mlx = Mlx()
        self.mlx_ptr = self.mlx.mlx_init()
        self.win = self.mlx.mlx_new_window(
            self.mlx_ptr, w, h, "Balle")
        self.img = self.mlx.mlx_new_image(self.mlx_ptr, w, h)
        self.data, _, self.sl, _ = \
            self.mlx.mlx_get_data_addr(self.img)

        # État physique
        self.t  = time.time()
        self.x, self.y   = w / 2, h / 4
        self.vx, self.vy = 150, 0

    def put_pixel(self, x, y, c):
        if 0 <= x < self.W and 0 <= y < self.H:
            o = y * self.sl + x * 4
            self.data[o:o+4] = c.to_bytes(4, 'little')

    def draw_circle(self, cx, cy, r, color):
        for dy in range(-r, r + 1):
            dx = int(math.sqrt(r*r - dy*dy))
            for i in range(-dx, dx + 1):
                self.put_pixel(cx + i, cy + dy, color)

    def update(self, dt):
        self.vy += self.GRAVITY * dt
        self.x  += self.vx * dt
        self.y  += self.vy * dt
        R = self.RADIUS
        if self.x + R >= self.W: self.x = self.W - R; self.vx *= -self.BOUNCE
        if self.x - R <= 0:      self.x = R;          self.vx *= -self.BOUNCE
        if self.y + R >= self.H: self.y = self.H - R; self.vy *= -self.BOUNCE
        if self.y - R <= 0:      self.y = R;          self.vy *= -self.BOUNCE

    def render(self, _):
        now = time.time()
        dt  = min(now - self.t, 0.05)  # dt max 50ms
        self.t = now
        self.update(dt)

        # Clear
        for i in range(0, self.sl * self.H, 4):
            self.data[i:i+4] = (0xFF0A0C10).to_bytes(4, 'little')
        self.draw_circle(int(self.x), int(self.y),
            self.RADIUS, 0xFF7C6AF7)
        self.mlx.mlx_put_image_to_window(
            self.mlx_ptr, self.win, self.img, 0, 0)

    def run(self):
        self.mlx.mlx_loop_hook(self.mlx_ptr, self.render, None)
        self.mlx.mlx_hook(self.win, 33, 0,
            lambda d: self.mlx.mlx_loop_exit(self.mlx_ptr), None)
        self.mlx.mlx_loop(self.mlx_ptr)

import time, math
from mlx import Mlx

class Waves:
    W, H = 800, 400

    def __init__(self):
        self.mlx = Mlx()
        self.mlx_ptr = self.mlx.mlx_init()
        self.win = self.mlx.mlx_new_window(
            self.mlx_ptr, self.W, self.H, "Ondes")
        self.img = self.mlx.mlx_new_image(
            self.mlx_ptr, self.W, self.H)
        self.data, _, self.sl, _ = \
            self.mlx.mlx_get_data_addr(self.img)
        self.start = time.time()

    def put_pixel(self, x, y, c):
        if 0 <= x < self.W and 0 <= y < self.H:
            o = y * self.sl + x * 4
            self.data[o:o+4] = c.to_bytes(4, 'little')

    def draw_wave(self, freq, amp, speed, phase, color, thick=2):
        t = time.time() - self.start
        for x in range(self.W):
            y = int(math.sin(x / freq + t * speed + phase) * amp + self.H / 2)
            for d in range(-thick, thick + 1):
                self.put_pixel(x, y + d, color)

    def draw_orbit(self, cx, cy, rx, ry, speed, color, r=8):
        t = time.time() - self.start
        ox = int(cx + math.cos(t * speed) * rx)
        oy = int(cy + math.sin(t * speed) * ry)
        for dy in range(-r, r + 1):
            dx = int(math.sqrt(max(0, r*r - dy*dy)))
            for i in range(-dx, dx + 1):
                self.put_pixel(ox + i, oy + dy, color)

    def render(self, _):
        # Clear
        for i in range(0, self.sl * self.H, 4):
            self.data[i:i+4] = (0xFF000000).to_bytes(4, 'little')
        # Ondes
        self.draw_wave(60, 80, 3, 0,    0xFF00E5A0, thick=2)
        self.draw_wave(70, 60, 2, 1.5,  0xFF7C6AF7, thick=1)
        # Orbe
        self.draw_orbit(self.W // 2, self.H // 2,
            120, 60, 2, 0xFFF7C26A)
        self.mlx.mlx_put_image_to_window(
            self.mlx_ptr, self.win, self.img, 0, 0)

    def run(self):
        self.mlx.mlx_loop_hook(self.mlx_ptr, self.render, None)
        self.mlx.mlx_hook(self.win, 33, 0,
            lambda d: self.mlx.mlx_loop_exit(self.mlx_ptr), None)
        self.mlx.mlx_loop(self.mlx_ptr)

    't_last': time.time(),
    't_next': time.time(),   # prochain rendu autorisé
    'fps': 0.0,
    'frames': 0,
    't_count': time.time(),  # fenêtre de comptage 1s
}

def render(s):
    now = time.time()

    # ── Limitation de FPS ──────────────────────
    if now < s['t_next']:
        return            # trop tôt : sauter cette frame
    s['t_next'] = now + FRAME_TIME

    # ── Calcul FPS réel (moyenne sur 1 seconde) ─
    s['frames'] += 1
    if now - s['t_count'] >= 1.0:
        s['fps'] = s['frames'] / (now - s['t_count'])
        s['frames'] = 0
        s['t_count'] = now

    # Afficher le FPS dans la fenêtre
    m.mlx_clear_window(mlx_ptr, win)
    m.mlx_string_put(mlx_ptr, win, 20, 40,
        0xFF00E5A0, f"FPS: {s['fps']:.1f}")
    m.mlx_string_put(mlx_ptr, win, 20, 70,
        0xFF7C6AF7, f"Target: {TARGET_FPS}")

m.mlx_loop_hook(mlx_ptr, render, state)
m.mlx_hook(win, 33, 0, lambda d: m.mlx_loop_exit(mlx_ptr), None)
m.mlx_loop(mlx_ptr)

import time
from mlx import Mlx

class FPSApp:
    TARGET_FPS = 60

    def __init__(self):
        self.mlx = Mlx()
        self.mlx_ptr = self.mlx.mlx_init()
        self.win = self.mlx.mlx_new_window(
            self.mlx_ptr, 400, 200, "FPS")
        now = time.time()
        self.t_next  = now          # prochain rendu autorisé
        self.t_count = now          # début de fenêtre de comptage
        self.frames  = 0
        self.fps     = 0.0
        self.frame_t = 1.0 / self.TARGET_FPS

    def should_render(self):
        # Retourne True si une frame doit être dessinée
        now = time.time()
        if now < self.t_next:
            return False
        self.t_next = now + self.frame_t
        return True

    def count_fps(self):
        now = time.time()
        self.frames += 1
        elapsed = now - self.t_count
        if elapsed >= 1.0:
            self.fps = self.frames / elapsed
            self.frames = 0
            self.t_count = now

    def render(self, _):
        if not self.should_render():
            return
        self.count_fps()
        self.mlx.mlx_clear_window(self.mlx_ptr, self.win)
        self.mlx.mlx_string_put(self.mlx_ptr, self.win,
            20, 40, 0xFF00E5A0, f"FPS: {self.fps:.1f}")
        self.mlx.mlx_string_put(self.mlx_ptr, self.win,
            20, 70, 0xFF7C6AF7, f"Target: {self.TARGET_FPS}")

    def run(self):
        self.mlx.mlx_loop_hook(self.mlx_ptr, self.render, None)
        self.mlx.mlx_hook(self.win, 33, 0,
            lambda d: self.mlx.mlx_loop_exit(self.mlx_ptr), None)
        self.mlx.mlx_loop(self.mlx_ptr)

from mlx import Mlx

class Colormap:
    def __init__(self, w=400, h=400):
        self.mlx = Mlx()
        self.mlx_ptr = self.mlx.mlx_init()
        self.w, self.h = w, h
        self.win = self.mlx.mlx_new_window(
            self.mlx_ptr, w, h, "Colormap")
        self.img = self.mlx.mlx_new_image(self.mlx_ptr, w, h)
        self.data, _, self.sl, _ = \
            self.mlx.mlx_get_data_addr(self.img)

    def put_pixel(self, x, y, r, g, b):
        offset = y * self.sl + x * 4
        color = 0xFF000000 | (r << 16) | (g << 8) | b
        self.data[offset:offset+4] = color.to_bytes(4, 'little')

    def render(self):
        for y in range(self.h):
            for x in range(self.w):
                self.put_pixel(x, y,
                    int(255 * x / self.w),
                    int(255 * y / self.h),
                    int(255 * (1 - (x + y) / (self.w + self.h))))
        self.mlx.mlx_put_image_to_window(
            self.mlx_ptr, self.win, self.img, 0, 0)

    def run(self):
        self.render()
        self.mlx.mlx_hook(self.win, 33, 0,
            lambda d: self.mlx.mlx_loop_exit(self.mlx_ptr), None)
        self.mlx.mlx_loop(self.mlx_ptr)
        self.cleanup()

    def cleanup(self):
        self.mlx.mlx_destroy_image(self.mlx_ptr, self.img)
        self.mlx.mlx_destroy_window(self.mlx_ptr, self.win)
        self.mlx.mlx_release(self.mlx_ptr)

class SpriteApp:
    def __init__(self):
        self.mlx = Mlx()
        self.mlx_ptr = self.mlx.mlx_init()
        self.win = self.mlx.mlx_new_window(
            self.mlx_ptr, 800, 600, "Sprites")
        self.sprites = {}

    def load(self, name, path):
        # Chargement selon l'extension
        if path.endswith('.png'):
            img, w, h = self.mlx.mlx_png_file_to_image(
                self.mlx_ptr, path)
        else:
            img, w, h = self.mlx.mlx_xpm_file_to_image(
                self.mlx_ptr, path)
        self.sprites[name] = (img, w, h)

    def draw(self, name, x, y):
        img, _, _ = self.sprites[name]
        self.mlx.mlx_put_image_to_window(
            self.mlx_ptr, self.win, img, x, y)

    def run(self):
        self.draw('puffy', 100, 100)
        self.draw('icon', 300, 100)
        self.mlx.mlx_hook(self.win, 33, 0,
            lambda d: self.mlx.mlx_loop_exit(self.mlx_ptr), None)
        self.mlx.mlx_loop(self.mlx_ptr)
        self.cleanup()

    def cleanup(self):
        for img, _, _ in self.sprites.values():
            self.mlx.mlx_destroy_image(self.mlx_ptr, img)
        self.mlx.mlx_destroy_window(self.mlx_ptr, self.win)
        self.mlx.mlx_release(self.mlx_ptr)

Conseils & astuces

// bonnes pratiques

🐢 mlx_pixel_put est lent
Pour dessiner beaucoup de pixels, créez une image, remplissez son buffer, puis appelez mlx_put_image_to_window une seule fois.
🔒 Libérer dans l'ordre
Toujours libérer : images → fenêtres → mlx_release. Inverser l'ordre peut provoquer des crashs ou fuites mémoire.
📌 Garder les références
Le wrapper Python stocke les callbacks en interne. Ne pas les réassigner avant la fin de la boucle, sinon ils seront garbage-collectés.
🖥️ Taille de l'écran
Utilisez mlx_get_screen_size pour créer une fenêtre adaptée à la résolution du moniteur courant.
🔄 mlx_sync
Pour éviter de lire une image pendant qu'elle est rendue, appelez mlx_sync(mlx_ptr, Mlx.SYNC_IMAGE_WRITABLE, img) avant toute modification.
📐 Vérifier les limites
MLX ne vérifie pas les coordonnées x/y. Un pixel hors-limites peut provoquer un comportement indéfini ou un crash silencieux.

from mlx import Mlx

class App:
    def __init__(self, w=800, h=600):
        self.w, self.h = w, h
        self.mlx = Mlx()
        self.mlx_ptr = self.mlx.mlx_init()
        self.win = self.mlx.mlx_new_window(
            self.mlx_ptr, w, h, "Mon App")
        self.img = self.mlx.mlx_new_image(
            self.mlx_ptr, w, h)
        self.data, self.bpp, self.sl, _ = \
            self.mlx.mlx_get_data_addr(self.img)

    def put_pixel(self, x, y, color):
        # Vérification des bornes intégrée
        if 0 <= x < self.w and 0 <= y < self.h:
            offset = y * self.sl + x * (self.bpp // 8)
            self.data[offset:offset+4] = color.to_bytes(4, 'little')

    def render(self, dummy=None):
        # Dessiner sur self.img puis envoyer en une fois
        self.put_pixel(400, 300, 0xFFFF0000)
        self.mlx.mlx_put_image_to_window(
            self.mlx_ptr, self.win, self.img, 0, 0)

    def run(self):
        self.mlx.mlx_loop_hook(self.mlx_ptr, self.render, None)
        self.mlx.mlx_hook(self.win, 33, 0,
            lambda d: self.mlx.mlx_loop_exit(self.mlx_ptr), None)
        self.mlx.mlx_loop(self.mlx_ptr)

    def __del__(self):
        # Nettoyage automatique à la destruction de l'objet
        try:
            self.mlx.mlx_destroy_image(self.mlx_ptr, self.img)
            self.mlx.mlx_destroy_window(self.mlx_ptr, self.win)
            self.mlx.mlx_release(self.mlx_ptr)
        except: pass

app = App()
app.run()
